# 텍스트와 이미지를 통한 감정 분석 — 447:1 데이터 불균형 문제 해결 과정

한국어 텍스트의 7가지 감정을 분류하는 딥러닝 모델 구현, 극단적 클래스 불균형(447:1) 문제를 정의하고 해결 전략을 수립한 프로젝트

| 해결 문제 | 감정 클래스 간 극단적 불균형(행복 43,786건 vs 공포 98건 = 447:1) |

## 프로젝트 배경

사람의 감정은 텍스트와 이미지 두 가지 경로로 표현된다. 실제 감정 분석 서비스에서는 텍스트만이 아닌 멀티모달 정보를 활용해야 더 정확한 감정 인식이 가능하다.

한국어 감정 데이터셋으로 모델을 구현하는 과정에서 **행복(43,786건)과 공포(98건) 사이 447:1의 극단적인 클래스 불균형**을 발견했다. 이 상태로 학습하면 모델이 다수 클래스(행복)만 예측하고 소수 클래스(공포, 놀람)는 무시하는 편향 문제가 발생한다.

불균형 문제를 어떻게 정의하고, 어떤 전략으로 접근할 것인가가 이 프로젝트의 핵심 과제였다.

## 핵심 구현 내용

### 1. 447:1 불균형 문제 정의 및 정량화

**분석 과정:** 학습 전 클래스별 샘플 수를 시각화하여 불균형의 실제 규모를 확인했다.

**핵심 발견:**
- 행복(43,786)이 전체의 78.7%를 차지
- 가장 적은 공포(98)와의 비율이 447:1
- 이 상태로 학습하면 모델은 모든 입력에 "행복"을 예측해도 78% 정확도를 달성 가능 → 무의미한 모델이 됨

**왜 이것이 문제인가:** 감정 분석에서 중요한 것은 소수 클래스(공포, 놀람)의 정확한 탐지다. 다수 클래스 위주의 학습은 실제 서비스에서 사용 불가능한 모델을 만든다.

---

### 2. 10배 언더샘플링 전략

**구현:** 최소 클래스(공포, 98건)의 10배를 목표 샘플 수로 설정 (목표: 980건/클래스)

**전략 선택 이유:**
- 오버샘플링(SMOTE 등): 텍스트 데이터에서 합성 샘플 생성이 의미 없음
- 언더샘플링: 다수 클래스를 줄여 균형을 맞추는 현실적 방법
- 10배 기준: 공포(98) × 10 = 980을 목표로, 각 클래스를 최대 980건으로 제한

**결과:** 55,600건 → 6,318건으로 데이터 축소 (데이터 손실의 대가로 균형 확보)

---

### 3. RNN 기반 텍스트 분류 모델

**구조:** 단어 임베딩(128차원) → RNN 은닉 레이어(64차원) → 완전연결층(7 클래스)

**학습 설정:**
- 옵티마이저: Adam (lr=0.001)
- 손실 함수: CrossEntropyLoss
- 조기 종료: patience=2 (검증 손실 기준)
- 배치 크기: 64

**한계 인식:** 기본 RNN은 LSTM, Attention 없이는 긴 문장의 맥락을 제대로 포착하기 어렵다. 모델 구조의 단순함이 성능 한계의 한 원인임을 학습 과정에서 확인했다.

## 주요 트러블슈팅

### 문제 1: 447:1 불균형으로 인한 모델 편향

**문제:** 언더샘플링 없이 학습 시 모델이 모든 입력에 "행복"만 예측하는 편향 발생

**원인 분석:** CrossEntropyLoss는 각 샘플에 동일한 가중치를 적용한다. 데이터의 78.7%가 행복이므로 모델이 행복만 예측해도 손실이 빠르게 감소한다.

**해결 방법:** 10배 언더샘플링으로 클래스별 샘플 수를 최대 980건으로 제한하여 균형 조정

**결과:** 편향 문제 해소 → 7개 클래스 전체에 대한 예측 시도 가능해짐. 그러나 전체 데이터의 89%를 제거한 대가로 모델의 일반화 성능 저하

---

### 문제 2: 언더샘플링 후에도 성능 32%에 그침

**문제:** 균형을 맞춘 이후에도 테스트 정확도가 32%로 낮음

**원인 분석 (다층적 원인):**
1. **모델 용량 부족:** 기본 RNN은 문장의 장거리 의존성 학습 불가 → KoBERT, LSTM 대비 감정 표현 포착 능력 열세
2. **소수 클래스 절대 부족:** 공포(98건)는 언더샘플링 후에도 그대로 98건 → 학습 데이터 자체가 너무 적음
3. **어휘 필터링 손실:** 1회 등장 단어 37.5% 제거 → 감정 표현의 핵심인 희귀 어휘 손실

**결과:** 세 가지 원인이 복합적으로 작용. 불균형 처리만으로는 부족하고 **모델 아키텍처 개선(LSTM/Transformer)이 병행되어야** 한다는 교훈 도출

---

### 문제 3: 수동 예측 테스트에서 오분류 다수 발생

**문제:** "무섭다 이거" (공포) → 놀람으로 분류, "정말 싫어" (혐오) → 행복으로 분류

**원인 분석:** 공포(98건)와 놀람(220건)처럼 절대 샘플 수가 극히 적은 클래스는 언더샘플링 이후에도 학습 데이터가 부족하다. 또한 한국어 감정 표현은 동일 단어가 문맥에 따라 다른 감정을 나타내므로 단순 RNN으로는 포착이 어렵다.

## 배운 점

**기술적 인사이트**
- 데이터 불균형은 "모델을 더 좋게 만드는 것"으로는 해결되지 않는다. 데이터 수집 단계부터 균형을 고려해야 한다.
- 언더샘플링은 클래스 균형을 맞추지만 데이터 손실이 크다. SMOTE, 클래스 가중치 설정(class_weight), Focal Loss 등의 대안을 함께 탐색해야 한다.
- 정확도(Accuracy)는 불균형 데이터에서 무의미한 지표다. 반드시 Macro F1-Score를 함께 봐야 한다.

**실무적 인사이트**
- 문제 정의 단계에서 "클래스가 몇 개인가"뿐 아니라 "클래스별 분포는 어떤가"를 먼저 분석하는 습관이 중요하다.
- 32%라는 수치보다, 32%가 되는 이유를 세 가지 원인으로 분리하여 설명할 수 있다는 것이 진짜 학습이었다.

## 향후 개선 방향

- **Transformer 계열 모델 적용:** KoBERT, KLUE-RoBERTa로 문맥 이해 능력 대폭 향상
- **Focal Loss 도입:** 소수 클래스에 더 높은 손실 가중치를 부여하여 언더샘플링 없이 불균형 처리
- **데이터 증강:** 소수 클래스 문장에 동의어 교체, 역번역(Back Translation) 등 적용
- **이미지-텍스트 융합 완성:** 이미지 특징을 CNN으로 추출 후 텍스트 특징과 결합하는 멀티모달 아키텍처 구현
